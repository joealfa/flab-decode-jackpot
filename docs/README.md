## üìù Documentation Maintenance

### When to Update Documentation

- **Adding Features:** Update relevant docs (API, Architecture, etc.)
- **Breaking Changes:** Update all affected documentation
- **Bug Fixes:** Update troubleshooting sections if applicable
- **Performance Changes:** Update performance documentation
- **API Changes:** Always update API_REFERENCE.md

### Documentation Style Guide

- Use clear, concise language
- Include code examples
- Keep formatting consistent
- Update table of contents
- Add links between related sections
- Include diagrams where helpful
